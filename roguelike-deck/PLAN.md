# ローグライクゲーム開発計画

## コンセプト

- **ジャンル**：カードバトル型ローグライト
- **コアコンセプト**：2枚のカードを合成して作った自分だけの切り札を軸に、開始デッキ・敵図鑑・進化カード・鍛冶を組み合わせて戦うカードローグライト
- **グラフィック**：まずASCII → 後でドット絵に移行
- **プラットフォーム**：Webブラウザ（itch.ioでリリース）

> 詳細仕様は **[SPEC.md](./SPEC.md)** を参照。

## 本作の独自要素

| 要素             | 内容                                                        |
| ---------------- | ----------------------------------------------------------- |
| オリジナルカード | 既存カード2枚を合成し、自分だけの切り札を作る               |
| エネミースロット | オリジナルカードに敵由来の効果を装着する専用スロット        |
| エネミーオーブ   | 敵図鑑報酬で得られる、敵に由来する特殊効果パーツ            |
| 敵図鑑           | 敵ごとの情報と進行度を記録し、エネミーオーブを解放する      |
| 開始デッキ       | キャラクターの代わりに、ラン開始時の戦術方針を選ぶ（4種類） |
| 初期遺物         | 開始デッキごとの個性を補強する初期装備                      |
| 進化カード       | ラン中の使用回数や条件達成によって進化するカード            |
| 鍛冶             | カード特殊効果付与、または遺物変換を選べるノード            |
| ポーション       | ラン中に使える消耗品アイテム                                |
| 実績             | 条件達成を記録し、一部要素の解放条件にもなる長期目標        |
| 称号             | 実績や条件達成で獲得できる表示用コレクション                |
| 進行度解放       | 実績・図鑑・試練などに応じてカードや遺物を解放する仕組み    |
| 試練レベル       | クリア後に解放される高難度やりこみ要素                      |

## 設計の大原則

ロジックと描画を最初から分離する。

```typescript
interface Renderer {
  render(state: GameState): void
}

class AsciiRenderer implements Renderer { ... }  // 最初はこれ
class PixelRenderer implements Renderer { ... }  // 後でこれに差し替え
```

## ゲームルール仕様（MVP基準）

### プレイヤー初期ステータス（SPEC.md §6.2 / §30.1）

| パラメータ         |                  値 |
| ------------------ | ------------------: |
| 最大HP             |                 100 |
| エナジー           | 3（毎ターン全回復） |
| 基本ドロー         |                   5 |
| 初期デッキ枚数     |                  10 |
| 初期ゴールド       |                 100 |
| ポーション所持上限 |                   2 |

### 初期デッキ構成

| 種別                 | 枚数 |
| -------------------- | ---: |
| 開始デッキ固有カード |    9 |
| オリジナルカード     |    1 |

### 開始デッキ一覧（SPEC.md §7.3）

| 開始デッキ | 得意                   | 苦手                   | 初期遺物     |
| ---------- | ---------------------- | ---------------------- | ------------ |
| 均衡型     | 安定した攻防           | 爆発的なコンボ         | 古びた紋章   |
| 連撃型     | 低コスト、ドロー、手数 | 高防御敵、手札妨害     | 小さな歯車   |
| 守護型     | 防御、反撃、長期戦     | 火力不足、ブロック貫通 | ひび割れた盾 |
| 侵蝕型     | 毒、裂傷、弱体         | 短期決戦、状態耐性     | 黒い小瓶     |

### バトルルール（SPEC.md §13）

- ターン開始時：ブロック初期化 → 状態処理 → エナジー回復 → **5枚ドロー**
- 山札不足時：捨て札をシャッフルして山札に移し、不足分を補充
- ターン終了時：未使用手札はすべて捨て札へ（保持カード除く）
- エナジーは次ターンに持ち越さない

### マップ構成（SPEC.md §12）

| 要素       | 内容                                                               |
| ---------- | ------------------------------------------------------------------ |
| 構成       | 複数階層（第1〜第3 + 最終）                                        |
| ノード種類 | 通常戦闘・エリート戦・ボス戦・イベント・休憩・鍛冶・ショップ・宝箱 |
| ボス       | 各階層最終ノード                                                   |

## MVP完成基準（SPEC.md §33）

以下をすべて満たせばMVP完成とする。

- 開始デッキを選択できる
- 開始デッキごとの初期遺物が機能する
- オリジナルカードを1枚作成できる（素材2枚合成）
- オリジナルカードには空のエネミースロットがある
- 作成したオリジナルカードを初期デッキに入れてランを開始できる
- 敵図鑑ポイントを獲得できる
- 図鑑ポイント満タンでエネミーオーブを入手できる
- エネミーオーブをオリジナルカードに装着できる
- カードを使って敵と戦える
- 山札・手札・捨て札が正しく循環する
- 敵が行動予定に従って行動する
- 勝利後にカード報酬を選べる
- 選んだカードが次戦闘のデッキに入る
- ポーションを獲得できる
- ポーションを戦闘中に使用できる
- 進化カードが出現し、進化条件を満たすと進化する
- 進化カードの進行度が記録される
- スロット付きカードが出現する
- 内部相性タグによる軽い報酬補正が機能する
- 内部相性タグはプレイヤーに表示されない
- マップ上で次ノードを選べる
- 休憩所で最大HPの20%回復または強化ができる
- 鍛冶でカード鍛冶か遺物変換を選べる
- カード鍛冶で、空き特殊効果スロットを持つカードに鍛冶特殊効果を付与できる
- 遺物変換で、不要な遺物2つを消費して新しい遺物を得られる
- 実績の進行度を内部的に記録できる
- 解放済みカード・遺物・ポーションを管理できる
- ボスを倒すとランが終了する
- 敗北するとリザルトに移動する
- ラン終了後、オリジナルカードを調整して再挑戦できる
